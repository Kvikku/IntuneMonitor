## Key Design Decisions

### Dual-Mode Entry Point

`Program.cs` checks `args.Length`:
- **No arguments** → Launches the interactive menu (`InteractiveMenu`)
- **With arguments** → Routes through `System.CommandLine` for direct CLI execution

Both paths share the same `AppConfiguration`, command classes, and logger factory.

### Storage Abstraction

`IBackupStorage` abstracts the backup backend:

```
IBackupStorage
├── LocalFileStorage   (writes JSON to timestamped directories)
└── GitStorage         (writes JSON + git commit + push)
```

`BackupStorageFactory` selects the implementation based on `Backup.StorageType`.

### Content Type Registry

`IntuneContentTypes` is the central registry mapping each content type to:
- Its Microsoft Graph API endpoint
- Its JSON backup filename
- Its backup folder name

Adding a new content type only requires adding entries to these dictionaries.

### Comparison Engine

`PolicyComparer` performs the diff between live and backup data:

1. **Index** both sets by policy ID
2. **Added** = IDs in live but not backup
3. **Removed** = IDs in backup but not live
4. **Modified** = IDs in both, with deep JSON property comparison

The comparer ignores volatile fields (`lastModifiedDateTime`, `version`, `@odata.type`) and handles assignment arrays with special human-readable formatting.

### Reporting Pipeline

Each command can produce:
- **Console output** — structured logging via `ILogger` + styled tables via `ConsoleUI`
- **JSON reports** — serialized `ChangeReport` / `ExportReport` objects
- **HTML reports** — self-contained dashboards generated by `HtmlReportGenerator` / `HtmlExportReportGenerator`

## Testing

Tests live in `tests/IntuneMonitor.Tests/` and use xUnit:

```bash
dotnet test
```

`PolicyComparerTests` validates the diff engine with scenarios for added, removed, and modified policies, field-level changes, and assignment handling.
